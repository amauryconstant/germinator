# Configuration

This directory stores configuration schemas, templates, and adapter configurations.

## Structure

- `schemas/` - Schema definitions for configuration files
- `templates/` - Configuration templates for different environments or use cases
- `adapters/` - Adapter-specific configuration files

## When to Add Files

Add files to these directories when implementing features that require configuration. For example:
- Define schemas in `schemas/` when adding new configuration options
- Add templates in `templates/` when supporting different environments
- Create adapter configs in `adapters/` when integrating external systems

## Notes

- Subdirectory READMEs will be added when content is added
- Focus on minimal, schema-based configuration
